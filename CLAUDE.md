## What the Robot Does

The robot follows a magnetic field generated by a guide wire laid on a surface. A signal generator (function generator or 555 timer) injects an AC signal into the wire, creating a changing magnetic field. The robot uses pickup inductor sensors to detect this field and steer itself along the wire.

**Two modes of operation:**
- **Manual mode:** Controlled via IR remote (infrared LED transmitter → IR receiver on robot)
- **Auto mode:** Robot autonomously follows the guide wire, handles intersections using a pre-programmed path table, and detects/avoids obstacles using VL53L0X ToF sensors

## Hardware Overview

### Microcontrollers (two required, different families)
- **Robot:** STM32L051 or STM32F051 (ST Microelectronics ARM family)
- **Remote:** EFM8LB12F64 (Silicon Labs 8051 family)

### Robot Subsystems
| Subsystem | Key Components | Interface |
|-----------|---------------|-----------|
| Motors (x2) | Solarbotics GM4 + MOSFET H-bridge + LTV-847 optocouplers | 4 PWM pins (2 per motor) |
| Field detectors (x3) | 1mH inductor + tank circuit + LM358 op-amp + MBR150 peak detector | 3 ADC inputs |
| Collision detection | VL53L0X Time-of-Flight laser modules (x2) | I2C (SDA, SCL) |
| IR receiver | TSOP33338 (38kHz demodulating receiver) | 1 digital input |
| Power | 4xAA (6V) → LM7805 (5V) → MCP1700 (3.3V) | — |

### Remote Subsystems
| Subsystem | Key Components | Interface |
|-----------|---------------|-----------|
| IR transmitter | SFH 4546 IR LED + 100Ω resistor | 1 digital/PWM output (38kHz modulated) |
| User input | Push buttons or keypad | Digital inputs |
| Joystick (optional) | Analog stick | 2 ADC inputs |
| Power | 9V battery → LM7805 (5V) → MCP1700 (3.3V) | — |

### Critical Power Notes
- Motor ground and logic ground must be **isolated** — use optocouplers (LTV-847) between the MCU and H-bridge
- Field detectors run on **5V** (op-amp needs headroom)
- MCU and VL53L0X run on **3.3V**
- MBR150 diodes used for USB/battery power switching
- Always add decoupling caps per LM7805 and MCP1700 datasheets

## Field Detection — How It Works

Each field detector circuit:
1. **Tuned tank circuit** — 1mH inductor + capacitor, resonant frequency 15–17kHz: `f_r = 1 / (2π√(LC))`
2. **Non-inverting amplifier** — LM358 op-amp, gain 10–20: `A_v = 1 + R1/R2`
3. **Peak detector** — MBR150 diode + RC, time constant τ >> 1/f_r, k = R3×C2 where 100 < k < 200

The robot has **3** of these circuits:
- **Left detector (d1)** — mounted on one side of the robot
- **Right detector (d2)** — mounted on the other side
- **Intersection detector** — mounted perpendicular, detects when the robot crosses another wire

### Tracking Logic
- `d1 > d2` → steer toward d1 side (closer to wire on that side)
- `d2 > d1` → steer toward d2 side
- `d1 ≈ d2` → go straight (centered on wire)
- Intersection detected → follow pre-configured path table

## IR Remote Control

- **Transmitter:** SFH 4546 IR LED, modulated at **38kHz**
  - Option A: Use a 555 timer as external 38kHz oscillator, gate it with MCU pin
  - Option B: Use MCU timer ISR to generate 38kHz carrier directly on a pin
  - A 38kHz UART TXD signal works as a simple transmitter
- **Receiver:** TSOP33338 — output can connect to RXD pin of a second UART
- Range needed: 1–2 meters
- Wire IR LED like a regular LED with 100Ω series resistor
- Tip: phone camera can "see" IR light to verify transmitter is working

### Required Commands
1. Stop
2. Forward
3. Backward
4. Turn Left
5. Turn Right
6. Rotate 180°
7. Select/configure pre-configured path
8. Start/stop auto mode

## Pre-Configured Paths

In auto mode, the robot follows the guide wire and makes decisions at intersections based on a stored path table. The guide wire is shaped like a figure-8 for testing, creating 8 intersections.

| Intersection | Path 1  | Path 2  | Path 3  |
|-------------|---------|---------|---------|
| 1           | Forward | Left    | Right   |
| 2           | Left    | Right   | Forward |
| 3           | Left    | Left    | Right   |
| 4           | Forward | Right   | Left    |
| 5           | Right   | Forward | Right   |
| 6           | Left    | Forward | Left    |
| 7           | Right   | Stop    | Forward |
| 8           | Stop    | —       | Stop    |

The user selects which path (1, 2, or 3) via the remote before starting auto mode.

## H-Bridge Motor Driver

- One H-bridge per motor (2 total)
- Each H-bridge: 1 P-MOSFET + 1 N-MOSFET per side (4 MOSFETs total per bridge)
- LTV-847 quad optocoupler isolates MCU signals from motor power
- R_D = 1kΩ, R_T = 10kΩ
- CW rotation: activate top-left P-MOSFET + bottom-right N-MOSFET
- CCW rotation: activate top-right P-MOSFET + bottom-left N-MOSFET
- **Never turn on both MOSFETs on the same side** — this shorts VDD to ground

## Coding Conventions

- All code in **C** (no Arduino, no C++)
- EFM8 toolchain: Simplicity Studio or Keil
- STM32 toolchain: STM32CubeIDE or arm-none-eabi-gcc
- Comment code thoroughly — Khushaan is learning and needs to understand every section
- Organize code into logical files: `main.c`, `motor.c`, `field_detector.c`, `ir_remote.c`, `vl53l0x.c`, etc.
- Use `#define` for pin assignments and magic numbers
- State machines should be explicit `switch/case` with named enum states
